# Prompt Shelf Web 项目规则

## 数据真源

- 本仓库只是真实网页代码的公开真源，不是任何人的私人数据真源。
- `main` 只保存框架、UI、空白 `data/prompts.json` 和位于 `templates/private-workspace/` 的空白 Skill 初始化模板；仓库根目录不得保存 `data/skills-index.json`。
- 用户主动连接的一个私有个人工作区，是该用户在 Prompt Shelf 中跨设备读取和写入提示词、Skill 索引与 AI 对话的唯一真源。
- 页面内的提示词和 Skill 必须共用同一个私有工作区连接；不得偷偷读取第二个仓库、旧分支、旧 HTML 或本机路径。
- 旧公开数据分支、旧 iCloud HTML 和旧 JSON 只能作为迁移或回滚证据，不能继续作为默认入口或同步目标。

## 公私边界

- 匿名访问和 Fork 默认只能看到空白工作台。
- 公开仓库、GitHub Pages 构建产物和历史不得包含真实提示词、完整 Skill、私人工作流、本机绝对路径、个人配置、Token 或 API Key。
- 未来分享只能使用明确的选择性“导出公开版”流程；不得改变或自动发布私有主库。
- 公共提示词数据和私有工作区初始化模板必须保持 0 条提示词、0 个 Skill。

## 私有工作区连接

- 优先使用浏览器 GitHub Contents API，不引入 Supabase、本地服务、Node 代理、启动器或必须先运行的后台进程。
- 只允许连接用户明确选择且拥有写权限的私有仓库。
- Fine-grained Token 只存当前设备 IndexedDB，使用 v2 私有工作区密钥；不得自动迁移旧公开分支 Token。
- IndexedDB 只按网页 origin 隔离，不按 GitHub Pages 项目路径隔离；不得把路径命名误述为安全边界，也不得在同一 Pages origin 部署不受信任页面。
- 提示词默认路径是 `data/prompts.json`，Skill 默认路径是 `data/skills-index.json`，AI 对话默认路径是 `data/ai-chats.json`，三者共享 owner、repo、branch 和 Token。
- 页面打开、恢复联网、重新聚焦、定时检查和手动同步时读取最新 SHA。
- 页面刷新后必须先验证目标仓库仍为 Private 且 Token 仍有写权限，再解锁私人缓存和 AI 配置；网络失败只能进入明确标注的离线缓存只读状态。
- 没有工作区身份的旧缓存必须按 legacy 缓存锁定，验证私有仓库并由用户明确确认后才能恢复。
- 本地 Skill 未提交并推送时不得显示为已同步；GitHub Pages 不因私人数据更新而重新部署。

## 同步和恢复

- 提示词写入前核对远端 SHA；冲突时重新读取并合并，不得用旧缓存直接覆盖。
- 永久删除和类目删除使用墓碑与恢复事件，避免旧设备复活数据。
- 浏览器有未同步修改时，不得用公共空白模板覆盖。
- Skill 索引只读；索引读取失败必须显示错误，不得将静态空白或旧缓存伪装成最新私有数据。
- AI 对话使用独立文件 SHA 和删除墓碑；空白草稿对话不写入远端，同一对话冲突时按消息 ID 合并，删除墓碑始终优先。
- 断开设备连接不得删除远端内容。

## 功能保护

- 保留提示词搜索、分类、标签、总助、CRUD、回收站、批量管理、GitHub 同步、AI 设置、AI 共创、AI 优化、AI 排版、版本历史、撤销恢复和移动端响应式布局。
- Skill 模式保留分类、搜索、卡片、详情、使用场景、引用说明、更新时间、状态和私有提示。
- 提示词和 Skill 共用同一套 AI 设置与 AI 导航助手；聊天记录可重命名、删除并跨设备同步。
- 提示词保存后可自动生成“它能做什么”和“适用场景”，但不得自动改写提示词正文。
- Agent 整理分类只允许调整元数据，不能删除、增减、改写或合并提示词正文；必须先预览再应用。

## 凭据红线

- GitHub Token 和 AI API Key 只保存在当前设备 IndexedDB。
- 凭据不得进入 HTML、Git、JSON、localStorage、导出、日志、截图、URL 或报错正文。
- AI 聊天输入和模型返回若包含疑似凭据，必须在写入聊天状态、localStorage 或私有仓库前 fail closed。
- 不得在公共代码中硬编码私人仓库名称、私有远程地址或本机绝对路径。

## 修改和验证

- 不回滚用户未提交修改，不使用 `git add -A`。
- 修改前读取本文件、`README.md` 和相关实现；保持改动可回滚。
- 发布前运行 `node scripts/verify-public-boundary.mjs`、内联 JavaScript 语法检查和 `git diff --check`。
- 手机端不得整体缩放或横向溢出；输入字号不得触发 iOS 聚焦放大，弹窗和长列表必须在视口内滚动。
- 远程推送、Pages 发布、删除旧分支、历史清理和私人数据迁移都需要明确确认。
- 最终状态必须区分：本地通过、已推送未发布、已发布已验收、仍有阻塞。
